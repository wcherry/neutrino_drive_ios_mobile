# Manual Verification: QR Code Key Import

Branch: `feature/epic-3-qr-key-import`
Feature flag: `FeatureFlags.qrKeyScan` (in `NeutrinoDrive/Config/FeatureFlags.swift`)

## Prerequisites

- [ ] Feature flag `FeatureFlags.qrKeyScan` is set to `true` in `FeatureFlags.swift`
- [ ] App installed on a physical iOS 16+ device OR an iOS Simulator that supports DataScannerViewController
- [ ] A test QR code is available, generated by the Neutrino Drive web app (or produced using the `makeQRString` helper in `KeyQRDecryptServiceTests` for development)
- [ ] The PIN used to encrypt the QR code is known

## Steps to Verify

### Happy Path

1. Launch Neutrino Drive and navigate to Settings.
2. In the "Encryption Key" section, tap "Import Encryption Key" (the list row).
3. Verify the `KeyImportView` sheet opens showing two buttons: "Import Key File" and "Scan QR Code".
4. Tap "Scan QR Code".
5. Verify the `KeyQRImportView` sheet opens with the camera scanner active.
6. Point the camera at a valid Neutrino Drive key QR code.
7. Verify the scanner stops and the PIN entry screen appears (showing a lock icon, "Enter your PIN" heading, a secure field, and "Decrypt & Import" button).
8. Enter the correct PIN in the secure field.
9. Tap "Decrypt & Import".
10. Verify a `ProgressView` spinner appears while the Argon2id KDF + decryption runs (takes ~0.15s).
11. Verify the success screen appears: green checkmark icon, "Keys imported successfully (v1)" label.
12. Verify the sheet auto-dismisses after approximately 1.5 seconds.
13. Verify back on the Settings screen the "Encryption Key: Imported ✓" row appears.

### Wrong PIN

1. Follow steps 1–8 of the Happy Path but enter an incorrect PIN.
2. Tap "Decrypt & Import".
3. Verify the error screen appears: red X icon, "Import Failed" heading, message "Failed to decrypt the key payload. Check your PIN and try again."
4. Verify a "Try Again" button is visible.
5. Tap "Try Again" and verify the camera scanner reactivates.

### Invalid QR Code (Non-NeutrinoDrive QR)

1. Follow steps 1–5 of the Happy Path but scan a QR code that is NOT a Neutrino Drive key QR (e.g. a URL or arbitrary text).
2. After scan, the app attempts to parse the string and should show the error screen: "The QR code does not contain a valid key payload."
3. Verify "Try Again" reactivates the scanner.

### Unsupported Version

1. Construct or present a QR code with `{ "v": 99, "alg": "argon2id+xchacha20", "payload": "..." }`.
2. Verify the error message: "The QR code uses an unsupported version."

### Device Not Supported (Simulator that lacks DataScanner)

1. Run the app on an iOS Simulator or device where `DataScannerViewController.isSupported` returns `false`.
2. Open the `KeyQRImportView` sheet by tapping "Scan QR Code".
3. Verify the fallback message is shown: "QR scanning not supported on this device."
4. Verify "Use the 'Import Key File' option instead." guidance is shown.

### Feature Flag Off

1. In `NeutrinoDrive/Config/FeatureFlags.swift`, set `qrKeyScan = false`.
2. Build and run the app.
3. Navigate to Settings and tap "Import Encryption Key".
4. Verify the `KeyImportView` shows only the "Import Key File" button — the "Scan QR Code" button must NOT appear.
5. Verify the existing JSON file picker still works normally.
6. Reset `qrKeyScan = true` when done.

### Existing JSON Import Not Broken

1. Ensure `FeatureFlags.qrKeyScan` is `true`.
2. Open `KeyImportView` and tap "Import Key File".
3. Select a valid JSON key file.
4. Verify keys import successfully (this exercises the pre-existing code path to confirm it is unaffected).

## Expected Results

| Scenario | Expected outcome |
|---|---|
| Valid QR + correct PIN | Success screen, auto-dismiss, keys in Keychain |
| Valid QR + wrong PIN | Error: "Failed to decrypt the key payload." |
| Non-NeutrinoDrive QR | Error: "The QR code does not contain a valid key payload." |
| `v != 1` | Error: "The QR code uses an unsupported version." |
| `alg` not `argon2id+xchacha20` | Error: "The QR code uses an unsupported encryption algorithm." |
| DataScannerViewController not supported | Graceful fallback message, no crash |
| Feature flag off | "Scan QR Code" button hidden; JSON import unaffected |

## Rollback

Set `FeatureFlags.qrKeyScan = false` in `NeutrinoDrive/Config/FeatureFlags.swift` and rebuild.
This hides the QR entry point entirely without removing any code. No deployment of a new build
is required if the feature flag is backed by a remote config system in future.
